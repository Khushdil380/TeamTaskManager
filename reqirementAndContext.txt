1. Project Overview
Project Title: Team Task Manager

Project Type: Full-Stack Web Application

Objective: The purpose of the Team Task Manager is to help teams organize projects, assign tasks, monitor progress, and collaborate efficiently using role-based access control.

The system allows
- Admins to manage projects and teams
- Members to work on assigned tasks
- Real-time task tracking and monitoring
- Secure authentication and authorization





2. Problem Statement
In many organizations and college teams
- Task management is unorganized
- Communication gaps occur frequently
- Task ownership is unclear
- Project progress becomes difficult to track
- Manual tracking leads to missed deadlines

This project solves these issues by providing:
- Centralized task management
- Role-based project handling
- Clear task ownership
- Deadline monitoring
- Dashboard-based tracking


3. Proposed Solution
The proposed solution is a web-based Team Task Management System where:

Users can register and login securely
Admins can create projects and teams
Tasks can be assigned to members
Members can update task progress
Dashboards provide project insights
Overdue tasks are automatically highlighted
Access is controlled using user roles

6. Functional Requirements
6.1 Authentication Module
User Signup
User Login
Logout
Password Encryption
JWT Authentication
Explanation: Users must create accounts securely. Passwords should never be stored in plain text.

Validations
Email must be unique
Password minimum length
Strong password validation
Invalid login handling
APIs
POST /api/auth/signup
POST /api/auth/login


6.2 Role-Based Access Control (RBAC)
Admin

Add/remove members
Create tasks
Assign tasks
Update any task
View analytics


Member

View assigned tasks
Update task status

Explanation: RBAC ensures security and controlled access.
